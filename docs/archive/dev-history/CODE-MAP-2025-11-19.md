## 🔧 Backend Architecture

### Module Structure
Each module follows clean architecture:
```
module/
├── module.repository.ts    # Database access
├── module.service.ts       # Business logic
├── module.controller.ts    # HTTP handlers
└── module.routes.ts        # Route definitions
```

### Core Modules

#### 1. Auth (`src/modules/auth/`)
- **Purpose**: Authentication & authorization
- **Key Files**:
  - `auth.service.ts` - JWT token management
  - `auth.middleware.ts` - Auth guard
  - `auth.controller.ts` - Login/refresh endpoints
- **Endpoints**:
  - `POST /auth/login`
  - `POST /auth/refresh`

#### 2. Document Types (`src/modules/documentTypes/`)
- **Purpose**: Document classification
- **Key Files**:
  - `documentTypes.service.ts` - CRUD + numbering rule creation
  - `documentTypes.controller.ts` - HTTP handlers
- **Endpoints**:
  - `GET /document-types`
  - `POST /document-types` - Creates type + numbering rule
  - `PUT /document-types/:id` - Updates type + rule
  - `DELETE /document-types/:id`
- **Features**:
  - Auto-numbering pattern support
  - Category classification
  - Digital signing requirement flag

#### 3. Numbering (`src/modules/numbering/`)
- **Purpose**: Auto-numbering engine
- **Key Files**:
  - `numbering.service.ts` - Pattern generation
- **Pattern Variables**:
  - `{AUTO}` - Sequential number (001, 002...)
  - `{YEAR}` - Current year (2025)
  - `{MONTH}` - Current month (01-12)
  - `{TYPE}` - Document type code
- **Example**: `{TYPE}-{AUTO}/{MONTH}/{YEAR}` → `CV-001/11/2025`

#### 4. External Organizations (`src/modules/external-orgs/`)
- **Purpose**: Manage external entities
- **Categories**: Government, Supplier, Customer, Partner
- **Endpoints**:
  - `GET /external-orgs`
  - `GET /external-orgs/stats`
  - `POST /external-orgs`
  - `PUT /external-orgs/:id`
  - `DELETE /external-orgs/:id`

#### 5. Documents (`src/modules/documents/`)
- **Purpose**: Document management
- **Sub-services**:
  - `tags.service.ts` - Document tagging
  - `permissions.service.ts` - Access control
  - `versions.service.ts` - Version tracking
- **Endpoints**:
  - `GET /documents`
  - `POST /documents` - Upload with auto-numbering
  - `GET /documents/:id/tags`
  - `POST /documents/:id/tags`
  - `GET /documents/:id/permissions`
  - `POST /documents/:id/permissions`
  - `GET /documents/:id/versions`
  - `POST /documents/:id/versions`

#### 6. Users, Departments, Roles
- **Purpose**: RBAC system
- **Features**:
  - User management
  - Department hierarchy
  - Role-based permissions (27 permissions)

## 🔐 Authentication Flow

```
1. User submits login (email + password)
   ↓
2. Backend validates credentials
   ↓
3. Backend generates JWT tokens:
   - Access token (1 hour)
   - Refresh token (30 days)
   ↓
4. Frontend stores in localStorage
   ↓
5. Frontend includes token in all requests
   ↓
6. When access token expires:
   - Frontend auto-refreshes using refresh token
   - Gets new access token
   - Continues seamlessly
```

## 📡 API Request Flow

```
Frontend Component
  ↓
useAuth().fetchJson('/endpoint')
  ↓
Adds Authorization header
  ↓
Backend Route
  ↓
authGuard middleware (validates token)
  ↓
Controller (handles request)
  ↓
Service (business logic)
  ↓
Repository (database access)
  ↓
Response back to frontend
```

## 🎯 Key Patterns

### Backend

#### Error Handling
```typescript
// Throw ApiError for consistent error responses
throw ApiError.unauthorized("Invalid token", "INVALID_TOKEN");
throw ApiError.notFound("Document not found", "DOC_NOT_FOUND");
throw ApiError.badRequest("Invalid input", "INVALID_INPUT");
```

#### Async Routes
```typescript
// Always wrap async handlers
router.get('/', asyncHandler(controller.list));
```

#### Auth Guard
```typescript
// Protect routes
router.use(authGuard);
router.get('/protected', asyncHandler(controller.handler));
```

### Frontend

#### API Calls
```typescript
// Use fetchJson from useAuth
const { fetchJson } = useAuth();
const data = await fetchJson<Type>('/endpoint');
```

#### React Query
```typescript
// For data fetching
const { data, isLoading } = useQuery({
  queryKey: ['key'],
  queryFn: () => fetchJson<Type>('/endpoint'),
});
```

#### Mutations
```typescript
// For create/update/delete
const mutation = useMutation({
  mutationFn: (data) => fetchJson('/endpoint', {
    method: 'POST',
    body: JSON.stringify(data),
  }),
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['key'] });
  },
});
```
